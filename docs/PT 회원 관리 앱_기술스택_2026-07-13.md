## 1. 개요 및 스택 선정 원칙
1인 개발자가 **실제 앱(iOS/Android)으로 무조건 출시**해야 하고, **기획 단계에서 팀원들에게 작동하는 화면을 가장 빠르게 프로토타이핑**해야 한다는 제약 조건을 반영하여 기술 스택을 확정했습니다.

이를 위해 **React Native (Expo) + Supabase** 조합을 단일 최종 스택으로 선정합니다.

1. **Expo Go를 통한 실시간 프로토타이핑**: 빌드나 배포 과정 없이 개발 중인 화면을 QR 코드 스캔만으로 팀원들의 실제 스마트폰(iOS, Android)에서 즉시 구동해 볼 수 있습니다.
2. **크로스플랫폼 앱 개발**: 하나의 TypeScript/React 코드로 iOS 앱스토어와 Google 플레이스토어에 동시 출시가 가능합니다.
3. **BaaS (Backend as a Service) 활용**: 백엔드 서버 없이 Supabase를 사용하여 데이터베이스, 사용자 인증, 이미지/영상 스토리지를 해결합니다.

## 2. 최종 기술 스택 요약

| 구분 | 선정 기술 스택 | 비고 / 역할 |
| :--- | :--- | :--- |
| **프론트엔드 (앱)** | **React Native (Expo)** + TypeScript | iOS, Android 네이티브 앱 개발 및 UI 구현 |
| **스타일링** | **NativeWind (Tailwind CSS)** | 유틸리티 클래스 기반 빠른 UI 스타일링 |
| **백엔드 & DB** | **Supabase** (PostgreSQL, Auth, Storage) | 회원 정보, 일정 데이터베이스 및 눈바디 이미지 저장 |

| **공유 링크 (웹)** | **Expo Router (Web target)** 또는 **Next.js** | 회원이 로그인 없이 운동 히스토리를 확인하는 고유 웹 링크 |
| **예상 비용** | **월 0원** (앱스토어 등록비 제외: Apple $99/년, Google $25 단회) | Supabase/Vercel/Expo 모두 무료 티어 한도 내 사용 가능 |

## 3. 기획 단계의 '초고속 프로토타이핑' 전략

### 3.1. Expo Go를 이용한 실기기 데모 (팀원 공유용)
* **어떻게 작동하나요?**: 개발자의 로컬 PC에서 Expo 개발 서버를 실행한 후 생성되는 QR 코드를 팀원들이 각자의 휴대폰에 설치된 **Expo Go** 앱으로 스캔합니다.
* **장점**: 
  - 개발자가 복잡한 빌드(IPA/APK 생성 및 배포)를 하지 않아도 기획 단계의 UI와 인터랙션을 실제 앱 환경에서 실시간으로 확인할 수 있습니다.
  - 디자이너와 기획자가 터치감, 폰트 크기, 화면 전환 등의 UX 요소를 실제 스마트폰 기기에서 체험하며 빠른 피드백을 주고받을 수 있습니다.

### 3.2. 웹 프로토타입 동시 지원
* **Expo Router**를 사용하면 앱 화면 코드를 그대로 웹에서도 구동 가능합니다.
* 웹 URL(Vercel 등에 배포)을 팀원들에게 공유하여, PC나 브라우저에서 즉시 접속하여 화면을 눌러볼 수 있도록 지원합니다.

## 4. 상세 스택 아키텍처 및 선정 이유

### 4.1. React Native (Expo)
* **이유**: Swift/Kotlin으로 따로 개발할 필요 없이 JavaScript/TypeScript를 사용하여 앱을 동시 개발할 수 있어 1인 개발 리소스를 1/3 이하로 줄여줍니다.
* **EAS Update**: 출시 이후 앱스토어의 번거롭고 느린 심사 과정을 거치지 않고, 자바스크립트 버그 수정이나 간단한 UI 변경 사항을 실시간(무선 OTA)으로 유저에게 배포할 수 있어 1인 유지보수성에 극히 유리합니다.

### 4.2. Supabase
* **이유**: 회원 목록, PT 수업 세션, 운동 기록, 예약 캘린더 등 다양한 데이터를 안전하게 관리할 수 있는 **PostgreSQL DB**를 즉시 제공합니다.
* **Storage**: 눈바디 사진, PT 촬영 동영상을 저장하기 위한 클라우드 공간을 무료 요금제(1GB) 범위 내에서 사용 가능합니다.
* **Auth**: 회원가입, 트레이너의 비밀번호 찾기, 간편 로그인 기능을 추가 서버 코딩 없이 바로 연동합니다.

## 5. 1인 개발 유지보수 및 운영 전략

1. **상태 관리**: 
   - 복잡한 전역 상태 도구(Redux 등) 대신, 서버 데이터 동기화를 위해 **React Query (TanStack Query)** 또는 Supabase SDK 자체를 사용하고, 클라이언트 상태는 React Context나 **Zustand**와 같이 가볍고 유지보수가 쉬운 도구를 채택합니다.
2. **스타일링**: 
   - **Tailwind CSS (NativeWind)**를 활용해 별도의 CSS 파일 관리 없이 마크업 내부에서 즉시 UI 스타일링을 처리하여 개발 생산성을 끌어올립니다.
3. **모바일 차트/그래프**:
   - 회원의 운동 기록 추이 시각화를 위해 가볍고 유지보수가 편리한 차트 라이브러리(**React Native SVG Charts**)를 활용합니다.
4. **카카오톡 공유 연동**:
   - `window.Kakao` SDK 또는 간편한 Web Share API를 사용해 트레이너가 회원의 눈바디/기록 공유 링크를 복사하거나 카카오톡으로 다이렉트 전송할 수 있게 구현합니다.

## 6. 추천 로드맵

1. **1단계: Expo 프로젝트 생성 및 Expo Go 구동 테스트 (1일)**
   - 초기 보일러플레이트 세팅 및 팀원 대상 테스트 링크 제공.
2. **2단계: Supabase 데이터 모델링 설계 (1일)**
   - 회원 테이블, 일정 테이블, 기록 테이블, 이용권 테이블 정의.
3. **3단계: 핵심 UI 프로토타이핑 (1주)**
   - 캘린더, 일정 등록, 회원 기록 및 피드백 입력 UI 구현 및 실기기 피드백 수렴.
4. **4단계: 백엔드 API 및 Storage 연동 (1주)**
   - Supabase를 통한 실제 데이터 저장, 로그인 기능 연동.
5. **5단계: 앱스토어 및 플레이스토어 심사 등록 (1주)**
   - 개발자 계정 개설 후 최초 릴리즈 빌드 출시.
